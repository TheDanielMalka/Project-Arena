# ── Pinning policy ────────────────────────────────────────────────────────────
# All entries use `==X.Y.*` — pip's prefix match, equivalent to
# `>=X.Y.0, <X.(Y+1).0`. That blocks surprise breaking-minor upgrades on
# deploy while still letting security patches inside the same minor in.
#
# Known gap (follow-up): we do NOT commit a true lockfile (hash-pinned
# patch + transitive deps). The Dockerfile should run `pip install` and
# `pip freeze > /app/requirements.lock.txt` in CI so each image records
# the exact resolved set; that lockfile can then be used with
# `pip install --require-hashes -r requirements.lock.txt` for
# reproducible rebuilds. Out of scope for this commit because it requires
# a working Python + network to resolve and verify.
